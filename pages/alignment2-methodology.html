Unified Reasoning Taxonomy Alignment

This document is the current method spec for mapping activation dimensions from several medical-reasoning models into one shared taxonomy. It should describe the selected pipeline, the active artifacts, and the checks that justify the current choices. It is not a history log.

Do not discard sentences before matching. Short filler sentences, setup sentences, and degenerate-looking transitions can carry trace-position and state-transition evidence.

Degenerate no-information sentences should be carved out first as an explicit reserved group. They remain in traces for transition modeling and path alignment, but they do not contribute semantic activation evidence to the ordinary dimension groups.

Pipeline

1. Standardize Activations

Let x[m,c,t,d] be the raw activation for model m, case c, sentence t, and dimension d. Standardize each dimension within its source model:

z[m,c,t,d]=(x[m,c,t,d]−mean[m,d])/std[m,d] z[m,c,t,d] = (x[m,c,t,d] - mean[m,d]) / std[m,d] z[m,c,t,d]=(x[m,c,t,d]−mean[m,d])/std[m,d]

Use signed z-scored activations for cross-model dimension correlations:

a[m,c,t,d]=z[m,c,t,d] a[m,c,t,d] = z[m,c,t,d] a[m,c,t,d]=z[m,c,t,d]

Use the absolute value of the final correlations during grouping, because two models may encode the same reasoning function with opposite activation direction.

For phase tracking, convert the standardized activation vector into a soft observation over states:

o[m,c,t]=softmaxd(z[m,c,t,d]) o[m,c,t] = softmax_d(z[m,c,t,d]) o[m,c,t]=softmaxd​(z[m,c,t,d])

The soft observation keeps shared sentence functions visible instead of forcing every sentence into a single hard state.

2. Build State And Estimate Phase From The Path

For each model, assign every sentence a hard state for counting and transition estimation:

h[m,c,t]=argmaxdx[m,c,t,d] h[m,c,t] = argmax_d x[m,c,t,d] h[m,c,t]=argmaxd​x[m,c,t,d]

Represent trace progress with 12 ordered bins over [0,1]. A sentence at position t in a trace of length T[c] lands at midpoint progress:

r[c,t]=(t+1/2)/T[c] r[c,t] = (t + 1/2) / T[c] r[c,t]=(t+1/2)/T[c]

Each dimension gets an empirical phase profile:

q[m,d,b]=Pr(progressbin=b∣hardstate=d) q[m,d,b] = Pr(progress_bin = b | hard_state = d) q[m,d,b]=Pr(progressb​in=b∣hards​tate=d)

If a dimension has no counts, use a uniform phase profile.

Estimate state-to-state transitions from adjacent hard states in the same trace. This transition matrix is the local path model:

A[m,i,j]=Pr(h[m,c,t]=j∣h[m,c,t−1]=i) A[m,i,j] = Pr(h[m,c,t] = j | h[m,c,t-1] = i) A[m,i,j]=Pr(h[m,c,t]=j∣h[m,c,t−1]=i)

For a fixed model m, row i describes which activation states usually follow state i. Rows are smoothed before normalization. This gives every transition nonzero support while preserving the observed path structure.

For each trace, filter the current observation through the previous state belief:

s[m,c,0]=normalize(o[m,c,0]) s[m,c,0] = normalize(o[m,c,0]) s[m,c,0]=normalize(o[m,c,0])

s[m,c,t]=normalize(o[m,c,t]∗transpose(A[m])s[m,c,t−1]) s[m,c,t] = normalize(o[m,c,t] * transpose(A[m]) s[m,c,t-1]) s[m,c,t]=normalize(o[m,c,t]∗transpose(A[m])s[m,c,t−1])

Equivalently, transpose(A[m]) s[m,c,t-1] predicts which states are plausible next from the path so far, and o[m,c,t] corrects that prediction with the current sentence's activation evidence. This is where path information enters phase estimation.

The sentence phase belief is the mixture of state phase profiles under the filtered state belief:

p[m,c,t,b]=sumds[m,c,t,d]q[m,d,b] p[m,c,t,b] = sum_d s[m,c,t,d] q[m,d,b] p[m,c,t,b]=sumd​s[m,c,t,d]q[m,d,b]

Phase is therefore derived from path-filtered state beliefs, not from a hand-written monotone mask over raw sentence index.

3. Score Sentence Pairs

For each shared case and model pair, score every cross-model sentence pair.

Text similarity uses L2-normalized text-embedding-3-small embeddings:

textsim(u,v)=max(0,dot(embedding[u],embedding[v])) text_sim(u,v) = max(0, dot(embedding[u], embedding[v])) texts​im(u,v)=max(0,dot(embedding[u],embedding[v]))

Phase similarity is cosine similarity between transition-filtered phase beliefs:

phasesim(u,v)=cosine(p[u],p[v]) phase_sim(u,v) = cosine(p[u], p[v]) phases​im(u,v)=cosine(p[u],p[v])

The sentence pair score is:

M[u,v]=textsim(u,v)∗phasesim(u,v) M[u,v] = text_sim(u,v) * phase_sim(u,v) M[u,v]=texts​im(u,v)∗phases​im(u,v)

4. Build Sentence Alignment Path Weights

Use path-local alignment weights. For each source sentence, find monotone target paths through the target trace and accumulate the local probability that the source sentence aligns to each target sentence. This uses the whole sentence-pair score matrix for the case, rather than only a small local anchor chunk.

Then apply reciprocal loopback repair. A sentence pair receives weight only when both directions agree that the pair is plausible:

W[u,v]=localweight(u−>v)∗localweight(v−>u) W[u,v] = local_weight(u -> v) * local_weight(v -> u) W[u,v]=localw​eight(u−>v)∗localw​eight(v−>u)

Normalize weights per case before accumulating dimension correlations.

This is separate from the path-filtered phase estimate above. Step 2 uses within-model state transitions to estimate each sentence's phase. Step 4 uses cross-model monotone paths to decide which sentence pairs should receive correlation weight.

Path-local repair is selected over the older anchor-chunk repair because it improved top-ranked judge precision in diagnostics while preserving the same loopback agreement principle.

5. Estimate Dimension Similarity

For every model pair and dimension pair, compute a weighted Pearson correlation over aligned sentence pairs:

corr(d1,d2)=weightedcorr(a[u,d1],a[v,d2],W[u,v]) corr(d1,d2) = weighted_corr(a[u,d1], a[v,d2], W[u,v]) corr(d1,d2)=weightedc​orr(a[u,d1],a[v,d2],W[u,v])

Store signed correlations in the pairwise artifact. Use abs(corr) when optimizing cross-model groups.

6. Carve Out Degenerate Sentences

Before optimizing semantic dimension groups, identify sentences whose trace function is no-information scaffolding: setup, transition, restatement, closure, formatting, or otherwise non-semantic connective text.

Assign these sentences to a reserved degenerate group:

group_index = -1
label = Degenerate No-Information

This group is part of the taxonomy state space, but not a recovered semantic dimension group. Keep its sentences in case traces so transitions into and out of no-information text remain visible. Exclude its sentence evidence from semantic group labeling and from the judge samples used to decide whether ordinary dimensions serve the same trace function.

The degenerate group is carved out before grouping so the optimizer is not forced to explain no-information text by merging unrelated dimensions into a residual semantic group.

7. Optimize Dimension Groups

The grouping target is a shared taxonomy where a dimension can belong to at most one group and a useful group can contain dimensions from multiple models. The next grouping pass should optimize the current three evaluation properties directly:

completeness:   every useful sample belongs to at least one group
consistency:    merged groups describe aligned trace functions
orthogonality:  a sample belongs to only one group

The operational objective should reward within-group pairwise evidence and penalize leaked evidence to outside dimensions. Grouping should not use fixed human anchor groups. The previous fixed U1/U7 anchors are diagnostics only.

Until the promoted pairwise artifact is regenerated and grouped, the complete no-fixed artifacts are provisional.

8. Label Groups

For each group, collect high-activation evidence sentences from all member dimensions. The labeler proposes a concise shared reasoning-function label and records supporting examples.

The judge system is organized around completeness, consistency, and orthogonality:

consistency:    do the sampled member dimensions serve the same trace function?
orthogonality:  is the sampled evidence better explained by only this group?
completeness:   does the taxonomy give each sampled trace function a home?

The critic should propose concrete changes that would improve those measures. After applying changes, resample evidence and repeat until the critic no longer identifies material improvements.

Do not ask the semantic labeler to name the reserved degenerate group. Its label is fixed by policy.

9. Project Sentences To Taxonomy States

After groups are labeled, project each sentence back onto the taxonomy.

For each sentence and group, aggregate member dimension activations. Assign the sentence to the best supported taxonomy group, preserving the source model, case, sentence index, text, and activation evidence.

If a sentence was carved out as degenerate, project it to the reserved degenerate group rather than to the highest-activation semantic group. This is not a late override that creates a new state after the taxonomy is finished; it is the projection of the first carved-out taxonomy group.

Regeneration Order

Run the selected pipeline in this order:

1. build per-model state machines
2. regenerate pairwise correlations with signed z-score transition path repair
3. identify and reserve the degenerate no-information group
4. regroup semantic dimensions without fixed anchors
5. relabel groups with completeness/consistency/orthogonality judging
6. resample and revise labels/groups until judge feedback converges
7. project sentences onto the final taxonomy, including group -1
8. rerun diagnostics and semantic scores

The method config in medinterp/alignment2/METHOD_CONFIG.json is the source of truth for artifact paths and selected runtime parameters.
