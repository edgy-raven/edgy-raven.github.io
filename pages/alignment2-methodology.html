Unified Reasoning Taxonomy Alignment

This document is the implementation plan for the alignment pipeline: how to map activation dimensions from several medical-reasoning models into one shared taxonomy. It is not meant to be an exhaustive derivation of standard numerical routines. It records the custom modeling choices, the order in which the real pipeline should run, and the failed experiments that explain why those choices are in the plan.

The selected implementation still lives in medinterp/alignment. This directory records the cleaned methodology and the artifacts selected from that workspace.

Pipeline

1. Normalize Activations

Normalize activations within each model and dimension. Let $x_{m,c,t,d}$ be the raw activation for model m, case c, sentence t, and dimension d:

μm,d=mean⁡c,txm,c,t,d,σm,d=std⁡c,txm,c,t,d,zm,c,t,d=xm,c,t,d−μm,dσm,d. \begin{aligned} \mu_{m,d} &= \operatorname{mean}_{c,t} x_{m,c,t,d},\\ \sigma_{m,d} &= \operatorname{std}_{c,t} x_{m,c,t,d},\\ z_{m,c,t,d} &= \frac{x_{m,c,t,d}-\mu_{m,d}}{\sigma_{m,d}}. \end{aligned} μm,d​σm,d​zm,c,t,d​​=meanc,t​xm,c,t,d​,=stdc,t​xm,c,t,d​,=σm,d​xm,c,t,d​−μm,d​​.​

If $\sigma_{m,d} = 0$, set it to 1. Use positive activations for pairwise alignment:

am,c,t,d=max⁡(0,zm,c,t,d). a_{m,c,t,d} = \max(0, z_{m,c,t,d}). am,c,t,d​=max(0,zm,c,t,d​).

Use absolute correlations later during grouping. A dimension may encode the same reasoning function with opposite activation direction across models, so anti-correlation is still alignment evidence.

2. Estimate Reasoning Phase Per Model

Build a per-model state model from the traces. The state model should estimate where each activation dimension tends to occur in reasoning progress, not just where a sentence appears by raw index.

For each model:

  1. Convert each sentence's normalized activation vector into a soft distribution over dimensions.
  2. Use those distributions across traces to estimate a posterior progress distribution for each dimension.
  3. For any sentence, define its phase distribution as the activation-weighted mixture of its dimension progress distributions.

The key requirement is that phase depends on the path through activated dimensions, not only on sentence index. Raw sentence index is too weak a proxy because traces have different lengths and models sometimes loop back to earlier reasoning functions.

3. Score Sentence Pairs

For each shared case and model pair, score every cross-model sentence pair. Use text-embedding-3-small sentence embeddings for text similarity and the state model for phase similarity.

Embeddings are L2-normalized. Text similarity is clipped cosine similarity:

sim⁡text(u,v)=max⁡(0,e^u⊤e^v). \operatorname{sim}_{\text{text}}(u,v) = \max(0, \hat e_u^\top \hat e_v). simtext​(u,v)=max(0,e^u⊤​e^v​).

Phase similarity is cosine similarity between posterior phase distributions:

sim⁡phase(u,v)=pu⊤pv∥pu∥∥pv∥. \operatorname{sim}_{\text{phase}}(u,v) = \frac{p_u^\top p_v}{\lVert p_u\rVert \lVert p_v\rVert}. simphase​(u,v)=∥pu​∥∥pv​∥pu⊤​pv​​.

The sentence-pair score is multiplicative:

Mu,v=sim⁡text(u,v)sim⁡phase(u,v). M_{u,v} = \operatorname{sim}_{\text{text}}(u,v) \operatorname{sim}_{\text{phase}}(u,v). Mu,v​=simtext​(u,v)simphase​(u,v).

The multiplication is intentional. It avoids letting a strong text match in the wrong reasoning phase, or a compatible phase match with unrelated text, dominate the alignment.

4. Build Sentence Alignment Weights

Use the match matrix $M$ to produce a soft alignment matrix for each case and model pair. The implementation should preserve three constraints:

  1. Allow 1:1, 1:many, and many:1 local alignments because traces have different sentence granularity.
  2. Avoid unconstrained many:many matching because it washes out sentence specificity.
  3. Allow loopbacks only when they are clearly better than the local monotone scaffold in phase-aware score.

Recommended implementation:

  1. Compute an optimal-transport warm start over the sentence-pair score matrix. This gives a soft global mass assignment without committing to a brittle hard path.
  2. Extract a monotone dynamic-programming scaffold from the warm start and match score. This scaffold represents broad trace progression.
  3. Form local anchor chunks from the scaffold. The chunks define the default local sentence-pair weights.
  4. Audit reciprocal nonlocal loopback candidates. A candidate may replace the local scaffold match only if it is reciprocal best among remaining candidates and improves both its row and column under $M_{u,v}$ and $\operatorname{sim}_{\text{phase}}(u,v)$.
  5. Normalize the final repair matrix to sum to one within the case.

The current artifact was generated with the repair-loopback anchor matcher without the optimal-transport warm start. The production implementation should add the warm start because the matching problem is the main source of alignment noise and the warm start makes the scaffold less dependent on local dynamic programming accidents.

5. Estimate Dimension Similarity

For each model pair, use the accepted sentence-pair weights to compute weighted correlations between activation dimensions. This is a standard weighted correlation calculation and does not need a custom implementation beyond respecting the sentence weights. Replace non-finite correlations with 0.

For grouping, use absolute correlation:

Ap,q=∣Rp,q∣. A_{p,q} = |R_{p,q}|. Ap,q​=∣Rp,q​∣.

Use the same alignment method across every model pair. Do not compare same-model dimensions when computing cross-model consistency.

6. Optimize Dimension Groups

Search for unified groups over all dimensions. Do not fix the number of groups and do not impose fixed U1/U7 anchors in the objective.

The selected objective uses:

correlation_mode = abs
orthogonal_floor = 0.0
orthogonality_objective = additive-leak
consistency_objective = all-pairs
score_objective = all-score-f1
consistency_score_weight = 1.0
group_count_penalty = 0.0
search_runs = 20
refine_passes = 2

Seed each candidate group with one close dimension per model where possible. Assign remaining dimensions by group affinity, then refine by moving dimensions between groups when the objective improves. Run several searches; the result should not depend on a single random seed.

All-pairs consistency asks whether the cross-model dimensions inside a group are mutually compatible, not just whether each dimension has one good mate:

C(G)=mean⁡p,q∈Gmodel⁡(p)≠model⁡(q)Ap,q. \begin{aligned} C(G) &= \operatorname{mean}_{\substack{p,q\in G\\ \operatorname{model}(p)\ne \operatorname{model}(q)}} A_{p,q}. \end{aligned} C(G)​=meanp,q∈Gmodel(p)=model(q)​​Ap,q​.​

The older member-best version was too permissive: a broad mixed group could score well if every dimension had one good partner, even when many other cross-model pairs in the same group were weak.

Additive-leak orthogonality penalizes separating dimensions that still have large cross-group alignment values. This is the pressure that keeps the optimizer from creating too many tiny high-consistency groups.

The group score is an F1-style combination of three summary scores: within-group consistency, cross-group orthogonality, and reciprocal-mate satisfaction. Reciprocal-mate satisfaction checks whether cross-model reciprocal-best dimension pairs are either in the same group or have an equally strong same-group alternative. Binary rogue-couple counts are reported as diagnostics, but they are not hard constraints because forcing every relative reciprocal pair together can make the smooth objective worse.

There is no separate size penalty. Group count emerges from the push-pull between consistency, additive leakage, and reciprocal-mate satisfaction.

7. Label Groups

Generate labels after grouping, not during optimization. The labeler should see several rounds of evidence:

  1. high-positive examples for the group,
  2. medium activation-band examples,
  3. random examples from the group's assigned dimensions,
  4. contrast examples from nearby groups,
  5. original per-dimension descriptions when available.

Use feedback iterations with a judge loop. The generator should propose a label and description; the judge should check specificity, overbreadth, and confusability with neighboring groups; the generator should revise using that feedback.

Labels are descriptions of the current grouping, not fixed ontology terms. If the grouping changes, labels should be regenerated.

After labels are generated, sort the generated taxonomy steps by progress in the unified taxonomy space. For each original activation dimension, use the state-machine topological_progress value as the primary progress coordinate and transition_progress only as a tie-breaker. For a unified group, compute the sentence-count-weighted mean progress over its member dimensions. The displayed step numbers are then assigned in ascending group progress order. The reserved degenerate state is not a generated taxonomy step and is displayed after the sorted taxonomy groups.

8. Project Sentences To Taxonomy States

For trace views, assign each sentence to the group containing its strongest dimension. Then apply one reserved-state override: if the full-corpus strict LLM degeneracy artifact strict_llm_mini_degenerate_sentence_judgments.jsonl marks the sentence as no-information, project it to group_index = -1, labeled Degenerate No-Information.

The reserved state is not a generated taxonomy category. It is used for markup, copied prompt boilerplate, output-template text, and standalone empty filler such as Hmm., Let me think., Wait., or Stop..

The override is sentence-level. A mixed group should not be removed wholesale: template rows route to -1, while genuine transition sentences remain in their generated taxonomy group.

Important Failed Experiments

These experiments should remain as diagnostics or ablations, but not as the default method unless a new comparison promotes them again.

  • Fixed U1/U7 grouping constraints: useful for sanity checks, but they lowered the selected objective and forced human priors into the taxonomy.
  • Hard mutable group count: made the answer easier to read but hid whether the objective actually preferred more or fewer groups.
  • Positive-only grouping correlations: missed dimensions that appear to encode the same function in opposite activation directions.
  • Separate group-size penalty: duplicated the role of additive-leak orthogonality and made the objective harder to interpret.
  • Sentence filtering before alignment: removed short transitions that often mark phase.
  • IDF sentence weighting: overemphasized lexical rarity and did not address the main alignment failure mode.
  • Bootstrap stability: useful as a diagnostic, but not needed as part of the core methodology.
  • Old monotone-only sentence alignment: too rigid when traces loop back or split one reasoning step across several sentences.
  • Run compression inside traces: should remain off by default unless it proves a clear benefit. The simpler uncompressed sentence alignment is easier to audit.
  • Reciprocal top-k sparsification and hard sentence caps: created hidden thresholds without solving the underlying matching problem.
  • Transport grouping: not part of the selected grouping objective. Optimal transport is better used earlier as a sentence-alignment warm start.

Interpretation

The pipeline aligns dimensions by asking whether two model dimensions activate on sentences that are semantically similar, in compatible reasoning phases, and connected by a mostly monotone but loopback-aware sentence alignment. The grouping step turns those pairwise dimension similarities into a shared taxonomy by balancing within-group cross-model consistency against cross-group leakage.
